Leave Monitoring

Project Overview

A shared Excel-based Leave Monitoring System that replaces individual, disconnected leave-tracking files used by HR employees. It automates employee data entry, leave-day computation, and payroll-ready reporting, allowing HR to process leave requests and payroll cut-offs faster and with far fewer errors.

Business Problem

Each HR employee maintained a separate Excel file for leave tracking, which caused duplicate entries and inconsistent records. Computing actual leave days was also manual and error-prone, since HR had to individually exclude weekends and holidays for each employee based on their location. On top of that, preparing leave data for payroll cut-offs required manually cross-checking records against pay periods.

Solution

I built a centralized, shared Excel workbook where HR employees select an employee's name from a dropdown and their personal information auto-populates. Leave type and charge codes are also selected from dropdowns to reduce input errors, while a built-in calculation automatically excludes weekends and holidays when computing leave days. A password-protected data sheet ensures only supervisors can edit submitted records, while other HR employees can still view them. A dedicated Payroll worksheet uses Power Query to pull and transform leave data based on a selected pay period, giving HR a ready-to-use summary of leave days and pay status with a single click of a Refresh button. The system also includes a dashboard and leave-pattern views so HR and management can monitor trends without manual reporting.

Key Features
  • Dropdown-based data validation for Type of Leave and Charged To
  • Auto-populated employee details based on Full Name selection
  • Centralized, shared data entry accessible to all HR employees
  • Automatic exclusion of weekends and holidays in leave-day computation
  • One-click, Power Query-driven payroll summary filtered by pay period
  • Password-protected records, editable only by supervisors
  • Dashboard with leave participation, tenure, and department insights
  • Automatic flagging of leave as "With Pay" or "No Pay" per entry
  • Employee-level and company-wide leave pattern tracking (day-of-week, monthly trends)
Technologies Used
  • Excel Formulas
  • Power Query
  • VBA (form submission automation)
  • Data Validation
  • Password Protection
  • OneDrive (shared file access)
Challenges or Learnings

Since the file was shared on OneDrive with more than 5 HR employees accessing and editing it simultaneously, it sometimes caused merge conflicts. This pushed me to think more carefully about how shared Excel files behave under concurrent use, and how to structure access (like password-protecting the data sheet) to reduce conflicts and protect data integrity.

Outcome or Impact

Before this system, it took 5 HR employees to manually encode around 100 leave entries per payroll cut-off, with computation errors often reported to managers. Now, a single HR employee can encode 100 leave entries in one day without issues, and payroll-ready leave summaries are generated instantly via the Payroll worksheet instead of manual cross-checking. HR meetings have shifted from resolving errors to simply giving upper management status updates.
